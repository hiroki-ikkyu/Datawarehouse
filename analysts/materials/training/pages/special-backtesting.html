バックテストの実践

推奨・組入の根拠を「過去に機能したか」で検証する

📖
バックテストとは

「もしこのシグナルに従って過去に売買していたら、どんなリターンだったか？」を検証すること。 素材アナリストとしては、例えば「鉄鋼メタルスプレッドが拡大し始めたら日本製鉄を買う」というルールが 過去10年で有効だったかを定量的にテストできる。

バックテストの3つの用途：
① 自分の投資シグナル（スプレッド/サイクル/バリュエーション）の有効性検証
② FMへのプレゼンで「過去にも機能した戦略」として説得力を持たせる
③ ペアトレードの過去パフォーマンスを確認してリスクを定量化

🏗
バックテストの5ステップ
基本フレームワーク
1
仮説（シグナル）を定義する
「何が起きたら買う/売るか」を明確なルールで定義。曖昧さを排除。
例：「LME銅在庫が3ヶ月移動平均を下回ったら住友鉱山をBuy」
↓
2
データを準備する
シグナルに必要なデータ（コモディティ価格、在庫、財務データ）と株価の時系列を揃える。
J-Quants API（日本株）＋Bloomberg/Trading Economics（コモディティ）
↓
3
エントリー/エグジットのルールを設定する
いつ買う（エントリー）、いつ売る（エグジット）、ポジションサイズ、保有期間。
例：「シグナル発生日の翌営業日にBuy。3ヶ月後にSell」
↓
4
シミュレーションを実行する
過去のデータでシグナルを機械的に適用し、各トレードのリターンを計算。
Python（pandas/numpy）またはExcelで実装
↓
5
結果を評価する
勝率、平均リターン、最大ドローダウン、シャープレシオ。
「ベンチマーク（TOPIX）に対してアルファが出ているか」が最終判断
📡
素材セクター向けシグナル例

① 鉄鋼メタルスプレッド反転シグナル

HRC価格と原料コスト（鉄鉱石×1.6＋原料炭×0.5）のスプレッドが底打ちから反転上昇し始めたら鉄鋼株をBuy。
ルール：
spread = HRC - (iron_ore * 1.6 + coking_coal * 0.5)
signal = spread の20日MAが60日MAを上回ったら → Buy 日本製鉄
exit = 20日MAが60日MAを下回ったら → Sell
期待：スプレッド拡大＝利益改善期に入るタイミングを捉える

② 石化スプレッド＄300割れからの回復

エチレン-ナフサスプレッドが$300（損益分岐）を割った後、$300を回復したらBuy。
ルール：
spread = ethylene_cfr_asia - naphtha_cfr_japan
signal = spread が$300以下 → $300を再度上回った日 → Buy 石化関連
exit = 90日保有 or spread が再度$300割れ
期待：赤字ゾーンからの脱出＝最悪期通過シグナル

③ LME在庫の方向転換シグナル

LME銅在庫が3ヶ月連続で減少（Phase 4→1への転換）したら住友鉱山をBuy。
ルール：
inv_change = LME_copper_inventory の月次変化
signal = inv_change が3ヶ月連続マイナス → Buy 住友鉱山
exit = inv_change が2ヶ月連続プラスに転じたら → Sell
期待：在庫減少＝需給タイト化→銅価格上昇→鉱山利益増

④ 中国クレジット・インパルス先行シグナル

中国のTSF（社会融資総量）の前年比成長率が加速に転じたら、6ヶ月後にコモディティ関連をBuy。
ルール：
impulse = TSF_yoy の3ヶ月MA の変化方向
signal = impulse がマイナス→プラスに転換 → Buy（6ヶ月後）コモディティバスケット
exit = impulse がプラス→マイナスに転換の6ヶ月後
期待：信用拡大→投資活発→素材需要増の因果チェーン（6-12ヶ月ラグ）

⑤ PBR 1x割れ＋配当利回り5%超

PBR 1.0x未満かつ配当利回り5%超の素材株をスクリーニングしてBuy。東証PBR是正の流れに乗る。
ルール：
signal = PBR < 1.0 AND div_yield > 5% AND Net_Debt/EBITDA < 3.0
→ 条件を満たす銘柄を等金額でBuy
exit = PBR > 1.0 に回復 or 12ヶ月保有
期待：バリュー＋インカムの二重の安全マージン。PBR是正がカタリスト

⑥ ミッドサイクルPEの割安シグナル

ミッドサイクルPE（過去10年平均EPSベース）が過去レンジの下位20%に入ったらBuy。
ルール：
mid_pe = price / avg_eps_10yr
signal = mid_pe < percentile_20(過去10年のmid_pe) → Buy
exit = mid_pe > percentile_50 → Sell
期待：サイクル調整済みの「本当の割安」を捉える

⑦ 鉄鉱石上昇で鉱山Long / 鉄鋼Short

鉄鉱石の20日MAが上昇トレンドに転じたら、BHP（又は住友鉱山）Long / JFE Shortのペアを組む。
ルール：
signal = iron_ore_20MA > iron_ore_60MA（ゴールデンクロス）
→ Long 住友鉱山 / Short JFE（ベータ調整済み）
exit = デッドクロス or 60日保有
期待：鉄鉱石の売り手（鉱山）にプラス、買い手（鉄鋼）にマイナスの裏表を利用

⑧ フジクラ/古河電工のスプレッド回帰

同業2社の株価比率（フジクラ÷古河電工）が過去平均から2σ以上乖離したら平均回帰を狙う。
ルール：
ratio = fujikura_price / furukawa_price
z_score = (ratio - ratio_mean_1yr) / ratio_std_1yr
signal = z_score > 2.0 → Short フジクラ / Long 古河電工
signal = z_score < -2.0 → Long フジクラ / Short 古河電工
exit = z_score が0に回帰
注意：構造的な乖離（フジクラのDC成長テーマ）が続く場合は回帰しない

実装方法
📝 Python バックテスト基本フレームワーク
import pandas as pd import numpy as np # === Step 1: データ読み込み === # J-Quants APIで日本株の日次終値を取得（既にセットアップ済み） # コモディティデータはCSVで用意（Bloomberg or Trading Economics） stock = pd.read_csv('nippon_steel_5401.csv', parse_dates=['date'], index_col='date') iron_ore = pd.read_csv('iron_ore_62fe.csv', parse_dates=['date'], index_col='date') coking_coal = pd.read_csv('coking_coal.csv', parse_dates=['date'], index_col='date') hrc = pd.read_csv('china_hrc.csv', parse_dates=['date'], index_col='date') # === Step 2: スプレッド計算 === df = pd.DataFrame() df['hrc'] = hrc['price'] df['raw_cost'] = iron_ore['price'] * 1.6 + coking_coal['price'] * 0.5 df['spread'] = df['hrc'] - df['raw_cost'] df['spread_ma20'] = df['spread'].rolling(20).mean() df['spread_ma60'] = df['spread'].rolling(60).mean() df['stock_price'] = stock['close'] # === Step 3: シグナル生成 === df['signal'] = 0 df.loc[df['spread_ma20'] > df['spread_ma60'], 'signal'] = 1 # Buy df.loc[df['spread_ma20'] < df['spread_ma60'], 'signal'] = 0 # Sell/Cash # === Step 4: リターン計算 === df['stock_return'] = df['stock_price'].pct_change() df['strategy_return'] = df['signal'].shift(1) * df['stock_return'] # 翌日リターン df['cum_strategy'] = (1 + df['strategy_return']).cumprod() df['cum_buyhold'] = (1 + df['stock_return']).cumprod() # === Step 5: 評価指標 === total_return = df['cum_strategy'].iloc[-1] - 1 sharpe = df['strategy_return'].mean() / df['strategy_return'].std() * np.sqrt(252) max_dd = (df['cum_strategy'] / df['cum_strategy'].cummax() - 1).min() win_rate = (df[df['signal'].shift(1)==1]['stock_return'] > 0).mean() print(f"Total Return: {total_return:.1%}") print(f"Sharpe Ratio: {sharpe:.2f}") print(f"Max Drawdown: {max_dd:.1%}") print(f"Win Rate: {win_rate:.1%}")
💡 shift(1)を忘れると未来のデータを使ってしまう
最も重要な注意点：シグナルにshift(1)を使って「翌日に売買する」想定にしないと、「今日のシグナルで今日の株価を売買した」ことになる（Look-ahead bias）。これをやると結果が圧倒的に良くなるが、現実では不可能な取引。バックテストの信頼性が完全に崩壊する。

Pythonが苦手ならExcelでも十分バックテストできる。構造は同じ。

列の構成：
A: 日付 ｜ B: 株価 ｜ C: 鉄鉱石 ｜ D: 原料炭 ｜ E: HRC
F: スプレッド =E-C*1.6-D*0.5
G: 20日MA =AVERAGE(F[直近20日])
H: 60日MA =AVERAGE(F[直近60日])
I: シグナル =IF(G>H, 1, 0)
J: 日次リターン =B2/B1-1
K: 戦略リターン =I1*J2（前日のシグナル×翌日のリターン）
L: 累積リターン =PRODUCT(1+K$2:K2)

評価：L列の最終値がトータルリターン。K列のAVERAGE/STDEVでシャープレシオ。

日本株の株価データ：

J-Quants API（推奨。既にセットアップ済み）

日次終値/出来高/調整済み株価。カバレッジ全45銘柄。無料プランで十分。Python連携可能。

コモディティデータ：

Bloomberg（6月〜）

BDH関数で日次/月次の全コモディティ時系列を取得。Excel直接連携。最も強力。

Trading Economics / Investing.com（無料）

主要コモディティ（鉄鉱石/銅/原油/金/BDI）の日次データをCSVダウンロード可能。遅延あり。

FRED（セントルイス連銀）

一部のコモディティ（原油WTI/銅/アルミ）＋マクロデータ（PMI等）。無料API。Python連携容易。

評価指標
トータルリターン
戦略全期間の累積リターン。Buy&Holdと比較。
(最終資産 / 初期資産) - 1
シャープレシオ
リスク調整後リターン。1.0以上で良好、2.0以上で優秀。
mean(return) / std(return) × √252
最大ドローダウン
ピークからの最大下落率。リスク管理の最重要指標。
min(累積リターン / 累積最大値 - 1)
勝率（Hit Rate）
プラスリターンのトレードの比率。55%以上が目標。
count(return > 0) / count(all trades)
Profit Factor
総利益 ÷ 総損失。1.5以上で有効な戦略。
sum(winning trades) / |sum(losing trades)|
アルファ
ベンチマーク（TOPIX）超過リターン。これが正でないと意味がない。
strategy_return - benchmark_return
⚠️
バックテストの7つの落とし穴
① Look-ahead Bias（未来データの使用）
最大の罠。「今日のスプレッドで今日の株を売買」はできない。シグナルは常に1日以上前のデータで生成し、翌日以降に売買する。決算データも発表日以降のみ使用可能。「四半期末のEPSで四半期中にトレード」は不可。
② Survivorship Bias（生存者バイアス）
上場廃止・倒産した銘柄がデータから除外されていると、「生き残った銘柄だけ」で良い結果が出る。素材セクターでは少ないが、過去の中小鉄鋼・非鉄メーカーの統合・上場廃止を考慮。
③ Over-fitting（過学習）
パラメータ（MA期間、閾値等）を過去データに最適化しすぎると、「過去にだけ機能する戦略」が出来上がる。

対処：
・パラメータは少なくする（2-3個以内）
・In-sample（前半7年）で構築→Out-of-sample（後半3年）で検証
・「なぜこのシグナルが機能するか」の経済的ロジックを必ず持つ
④ 取引コストの無視
売買手数料、スリッページ（大型株でも0.1-0.3%）、ショートの貸株コスト（年1-3%）を含めないとバックテストは楽観的すぎる。片道0.1-0.2%の取引コストを必ず差し引く。
⑤ 流動性の無視
小型の素材株（大和工業、共英製鋼等）は1日の出来高が少なく、大きなポジションを取ると市場インパクトが出る。バックテストでは「いくらでも買える」前提だが、現実は違う。出来高の5%以下のポジションサイズを前提にする。
⑥ レジーム変化（構造変化）
過去10年で有効だったシグナルが今後も有効とは限らない。中国の構造変化（不動産→ハイテク）、脱炭素、EV化でコモディティの需要構造が変わっている。「なぜ機能するか」の論理が今後も成立するかを常に問う。
⑦ 少ないサンプル数
月次シグナルで10年＝120データポイント。トレード回数が20回以下だと統計的有意性がない。 最低30トレードのサンプルが欲しい。足りなければ対象銘柄を増やすか、日次に切り替える。

実践ワークフロー

Step 1：中級編#1-6で学んだ分析手法から「バックテスト可能なシグナル」を1つ選ぶ

Step 2：J-Quants APIで株価データ、Trading EconomicsでコモディティデータをCSV取得

Step 3：PythonまたはExcelでシグナル→リターン計算を実装。最初は単純な戦略から。MAクロスオーバーで十分

Step 4：評価指標（シャープ/勝率/ドローダウン）を計算。Buy&Holdと比較

Step 5：In-sample/Out-of-sampleに分けて検証。Out-of-sampleでも機能するか確認

Step 6：結果が良ければ投資メモに「過去10年のバックテストで年率+X%のアルファ、シャープY」と記載

Step 7：Claude Codeで自動化。GitHub Actionsで毎日シグナルを更新→Morning Briefに組み込み

💡 最初にやるべきバックテスト（推奨）
「鉄鋼メタルスプレッドのMAクロス → 日本製鉄の株価」
理由：① データが比較的入手しやすい ② 経済的ロジックが明快（スプレッド改善→利益改善→株価上昇）③ 中級編#1のスプレッド分析の直接的な検証 ④ FMへの説得力が高い

これが機能すれば、同じフレームワークを銅/石化/石油精製に展開できる。
